# Интеллектуальная Система Составления Расписаний (RL Hyper-Heuristic)

Данный репозиторий содержит программный комплекс для автоматической генерации и оптимизации учебных расписаний масштаба факультета/университета. Система решает классическую задачу составления расписаний (University Course Timetabling Problem - UCTP) с использованием гибридных подходов и методов искусственного интеллекта.

## Особенности архитектуры

Проект реализует два различных подхода к генерации расписаний:

1. **Классический Гибридный Алгоритм (Static Hybrid):**
   * **Phase 1 (GA):** Генетический алгоритм для быстрого глобального поиска.
   * **Phase 2 (CP-SAT):** Математический решатель (Google OR-Tools) для 100% устранения жестких конфликтов (накладок, нехватки вместимости).
   * **Phase 3 (SA):** Имитация отжига (Simulated Annealing) для локальной оптимизации мягких ограничений (устранение "окон", минимизация поздних пар, учет пожеланий преподавателей).

2. **Обучение с подкреплением (RL Hyper-Heuristic):**
   * Вместо жестко заданного пайплайна используется нейросетевой агент (PPO из `sbx` — JAX-реализация алгоритмов от создателей Stable-Baselines3).
   * Агент самостоятельно выбирает, какую эвристику (GA, CP-SAT или SA) применить на каждом шаге оптимизации, адаптируясь под текущее состояние расписания.

## Структура проекта

* `src/models.py` — Pydantic модели данных (Teacher, Group, Room, Event, Slot).
* `src/solver.py` — Классический статический конвейер (HybridScheduler).
* `src/rl_solver.py` — Исполнитель на базе RL-модели (RLScheduler).
* `src/validator.py` — Строгая система проверок расписания и подсчета метрик качества.
* `src/data_generator.py` — Генератор реалистичных датасетов уровня факультета (~200+ занятий).
* `train_fast_rl.py` — Скрипт быстрого обучения RL-модели на симулированной среде.
* `run_benchmark.py` — Главный скрипт для сравнения производительности Static vs RL.
* `rl_models/` — Директория с обученными весами нейросетевых моделей.

## Установка и запуск

1. **Создайте и активируйте виртуальное окружение:**
   ```bash
   python -m venv venv
   source venv/bin/activate
   ```

2. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Запустите бенчмарк для проверки работоспособности:**
   ```bash
   python -m run_benchmark
   ```
   Этот скрипт сгенерирует набор данных (20 групп, 40 преподавателей, 25 аудиторий), решит задачу обоими методами и выведет сравнительную таблицу с метриками в консоль, а также сохранит отчет в `benchmark_results.csv`.

## Основные команды (Сценарии использования)

После настройки окружения вы можете запускать различные сценарии системы:

### 1. Генерация расписаний
* **Генерация с использованием гибридного алгоритма:**
  ```bash
  python test_large_run.py
  ```
  Создает большое расписание (100+ занятий), оптимизирует его методами GA -> CP-SAT -> SA и сохраняет результаты в текстовом, JSON и HTML форматах (`schedule_output.*`).

* **Генерация с использованием RL-агента:**
  ```bash
  python test_rl.py
  ```
  Использует обученную нейросеть для интеллектуального подбора эвристик оптимизации. Выводит метрики и генерирует файлы `rl_schedule_output.*`.

### 2. Сравнение и бенчмаркинг
* **Масштабный бенчмарк (уровень факультета):**
  ```bash
  python -m run_benchmark
  ```
  Генерирует датасет на 200 занятий (20 групп, 40 преподавателей) и запускает оба алгоритма последовательно, выгружая результаты в `benchmark_results.csv`.

### 3. Обучение RL-агента
* **Быстрое обучение (Симуляция):**
  ```bash
  python train_fast_rl.py
  ```
  Обучает агента (PPO) на симулированной `fast_env`, что занимает всего пару минут. Идеально для отладки функций вознаграждения. Модель сохраняется в папку `rl_models/`.

* **Полное обучение:**
  ```bash
  python train_rl.py
  ```
  Полноценное обучение агента на реальных вызовах GA, CP-SAT и SA с генерацией разнообразных учебных контекстов. Занимает больше времени.

## Ключевые метрики

Алгоритмы оцениваются по следующим параметрам:
* **Hard Conflicts:** Накладки аудиторий, преподавателей или групп. Система гарантирует их отсутствие (сводит к 0).
* **Group/Teacher Windows:** Количество "окон" (пустых пар между занятиями в один день).
* **Late Starts Penalty:** Штраф за позднее начало занятий (стимулирует проведение пар в утреннее время).
* **Teacher Satisfaction:** Удовлетворенность преподавателей (учет их пожеланий по дням и времени).
